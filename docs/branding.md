# Branding and Discoverability

DICOMForge is positioned as a lightweight Python library for medical imaging
developers who want typed DICOM metadata handling, explicit safety boundaries,
and optional integrations with the wider pydicom, NumPy, Pillow, pynetdicom,
and DICOMweb ecosystem.

## Naming

- Project name: `DICOMForge`
- Package name: `dicomforge`
- Import namespace: `dicomforge`
- Repository: `mustafamm072/DicomForge`

Use `DICOMForge` in prose and `dicomforge` only when referring to the Python
package, install command, or import namespace.

## Adoption Message

Recommended short description:

> A lightweight, typed Python DICOM processing toolkit for medical imaging.

Recommended longer description:

> DICOMForge gives Python medical imaging teams a small, predictable DICOM
> toolkit with typed tags, pixel safety checks, auditable de-identification,
> DICOMweb helpers, and optional adapters for pydicom, NumPy, Pillow, and
> pynetdicom.

## Commercial Positioning

DICOMForge should be described as a developer library, not as a diagnostic
viewer, PACS, medical device, or compliance certification. Commercial teams
should be pointed to the MIT license, the optional-dependency model, the
conformance notes, and the safety notes before regulated clinical use.
